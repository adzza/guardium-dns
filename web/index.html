Router integration

Pick the router family Guardium should push enforcement to. Different vendors support different stages — the toggles below adapt automatically.

Router (ASUS)

Connect your ASUS router to give the dashboard a true kill-switch. Devices on the Internet Off profile are added to the router's MAC-based block list, defeating hardcoded DNS and DoH bypasses (e.g. some Smart TVs).

Router clients

DNS Director

Forces every device on a managed profile (kids, no-streaming, no-YouTube, etc.) to use this dashboard for DNS — even if they hardcode 8.8.8.8 on the device. Devices on Internet Off are skipped (already L2-blocked).

DoH IP blocklist (SSH)

Connects to your router over SSH and adds iptables rules that drop traffic from managed devices to well-known DoH/DoT endpoints (1.1.1.1, 8.8.8.8, 9.9.9.9, etc.). Catches Smart-TV apps with embedded DoH clients that DNS Director alone can't stop.

Router (UniFi)

alpha

Connects to your UniFi Network controller (UDM, UCG, UniFi OS Server, or standalone Network Application). Stage 1 (kill switch) and Stage 3 (DoH / DoT block) use native Traffic Rules — no SSH required. Stage 2 (per-MAC DNS Director) is not supported by UniFi firmware.

Stage 2 caveat: UniFi firmware has no per-MAC DNS Director primitive. Use DHCP-level DNS pinning (LAN → DHCP DNS) or a separate VLAN to force kid-cohort devices onto Guardium's DNS, then rely on Stages 1 + 3 above for the hardening layer.

How the three stages stack up

  1. Stage 1 MAC-based "Internet Off" — the dashboard mirrors the Internet Off profile into the router's Block Internet Access list. Defeats hardcoded DNS / DoH / VPN attempts because the firewall drops everything at the MAC layer before it leaves the LAN.
  2. Stage 2 DNS Director (per-MAC redirect) — for any device on a managed profile (kids/no-streaming/no-YouTube/default), the router transparently rewrites all port-53 traffic to this dashboard, so a Smart-TV typing in 8.8.8.8 still gets filtered answers.
  3. Stage 3 DoH IP blocklist (SSH+iptables) — adds firewall drops from each managed MAC to a curated list of well-known DoH/DoT endpoints (Cloudflare, Google, Quad9, AdGuard, OpenDNS, etc.) on TCP/443, UDP/443 and TCP/853. Catches apps that embed their own DoH client.

Credentials are encrypted at rest. All router rules created by the dashboard are tagged so they can be cleanly removed without ever touching anything you've added by hand. Router push is best-effort — DNS-level enforcement keeps running even if the router is unreachable.
